Profesionele leerdoelen

Leerdoel 1: prototyping

Ik wil goede prototypes kunnan maken, waarbij ik goede bruikbare feedback goed verwerk in mijn projecten. Ook wil ik de gemaakte prototypes goed kunnen presenteren aan anderen, zodat ik betere en gerichte feedback kan ontvangen over het gemaakte werk.

Leerdoel 2: samenwerken

Ik wil goed kunnen samenwerken met anderen, waarbij ik goed kan communiceren en mijn ideeën duidelijk kan overbrengen. Ook wil ik goed kunnen luisteren naar anderen en hun ideeën serieus nemen, zodat we samen tot betere oplossingen kunnen komen.

Leerdoel 3: communiceren

Ik wil de gebruiker goed kunnen Oriënteren en begrijpen, zodat ik user interfaces kan maken die goed aansluiten bij de gebruiker(s). Op deze manier kan de website de pijnpunten van gebruikers omzeilen.

Weekly nerd

Killian valkhof

De code kan zo robuust mogelijk gemaakt worden door het gebruik van javascript te minimaliseren. Javascript bevat veel goede functionaliteiten die de gebruiker veel functies geeft om het web te bedienen. Als de gebruiker geen javascript in kan laden, dan lopen de gebruikers tegen het probleem aan dat de website niet meer optimaal werkt. Als oplossing kunnen zoveel mogelijk javascript functionaliteiten vermeden worden en de kracht van html gebruikt worden. Ook kan "noscript" als fallback gebruikt worden om de gebruiker alsnog een goede website belevening te bieden.

Peter Paul Koch

De browser bestaat al een lange tijd. En sinds het ontstaan van de browser zijn er vele aanpassingen en functies erbij gekomen. Zo was er vroeger alleen de mogelijkheid om html te gebruiken, waardoor er geen styling kon plaatsvinden. Later kwamen css en javasript erbij, met deze toevoegingen kon er veel meer bereikt worden op het web.
Vroeger had elke browser zijn eigen javascript engine om javascript te kunnen schrijven.
De rendering engine is verantwoordelijk voor het renderen van de html en css. De render constuctureerd de DOM. Dit maakt geen interface of javascript engine.
Als iets ooit heeft gewerkt in de browser, dan moet dit ook altijd blijven werken. Dit heet backwards compatibility.

Nils Binder

Designs moeten niet altijd flat zijn, maar ook bepaalde interactie bevatten om de gebruiker een ervaring mee te geven. De interacties kunnen op verschillende manier geoptimaliseerd worden voor de gebruiker. Grids en Subgrid kunnen hier een rol inspelen, zodat de elementen op de website goed geordend blijven. Ook tussen de pagina's kunnen animaties gebruikt worden. Denk hierbij aan een navigatie die van de ene actieve pagina naar de andere animeert. Hiermee wordt er voor een betere gebruikerservaring gezorgd.
Als er een error message wordt gegeven, dan is het belagrijk dat deze error message de volledige informatie bevat voor de gebruiker. Hiermee wordt de UX beter.
Als er een bepaalde veld (required) is om in te vullen, dan is het belangrijk dat de gebruiker weet wat de verschillende tekens betekenen op de website. Zo kan er optional gebruikt worden om dit te verduidelijken.

Rosa

Servers worden beheerd door verschillende partijen. De servers kunnen gehackt worden en zo vele verschillende data van veel mensen verkrijgen. Als je zelf een server opstelt en gebruikt, dan bezit je de data zelf. Hierdoor ben je niet afhankelijk van partijen die meekijken of hackers die veel data kunnen stelen. Ook de beveiliging is belangrijk voor een de server. Dit is waar "Planner Obsolence" zich mee bezig houdt. Het waarborgen van veiligheid.

The web you want

The web you want

The web you want richt zich vooral op de meningen en argumenten van verschillende gastsprekers. Zo willen de gastsprekers vooral dat het web accessible wordt en ook inclusief voor veel mensen. Ook worden er veel argumenten gegeven waarom het web toegankelijk en inclusief moet zijn. Zo is het web een belangrijk onderdeel van ons leven en is het belangrijk dat iedereen toegang heeft tot het web, ongeacht hun beperkingen of omstandigheden. Ook wordt er benadrukt dat het web een plek moet zijn waar iedereen zich welkom voelt en waar iedereen kan deelnemen aan de online gemeenschap.
